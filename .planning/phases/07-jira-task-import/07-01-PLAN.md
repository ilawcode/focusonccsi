# Plan 07-01: Stabilize Import & Waterfall Flow

Implement the bulk import logic, configurable mapping, and dashboard refined for waterfall tracking.

## User Decisions (Locked)
- **Mapping**: Dates mapped via `src/config/jira-date-mapping.json`.
- **Fields**: Key, Summary, Description, Type only (Jira status removed from UI).
- **Update Policy**: Metadata updates from Jira; Timeline dates stay local.
- **Assignment**: All imports auto-assigned to self.

## Tasks

### Wave 1: Foundation & Proxy Stability

1. **Create Mapping Config**
   - Create `src/config/jira-date-mapping.json`.
   - Action: Define a mapping object where keys represent the 14 local phase dates and values are empty or placeholder Jira field IDs (e.g., `customfield_12345`).
   - Acceptance: File exists and is valid JSON.

2. **Enhance Search Proxy for Metadata**
   - Update `src/app/api/jira/search/route.ts`.
   - Action: Ensure the `fields` array passed to Jira includes `description` and `issuetype`. (Key, summary, status are already there).
   - Acceptance: Grep `jiraFields` in `src/config/jira-fields.json` includes "description" and "issuetype".

### Wave 2: Bulk Import Logic

3. **Extend Import API for Bulk**
   - Modify `src/app/api/tasks/import/route.ts`.
   - Action:
     - Check if the request body contains `issues` (array) or `issue` (single).
     - Map over issues and perform `findOneAndUpdate` for each.
     - Inject date pre-filling logic using `jira-date-mapping.json` (if fields match).
   - Acceptance: API responds with a summary of imported/updated counts.

4.  **Implement "Import All" UI**
    - Modify `src/app/dashboard/page.tsx`.
    - Action:
      - Add "Import All" button next to "Search Jira".
      - Update `handleJiraSearch` to pass necessary metadata.
      - Update `importingId` to handle a bulk state.
    - Acceptance: Clicking "Import All" triggers a request to `/api/tasks/import` with all search result objects.

### Wave 3: Waterfall UI Refinement

5.  **Remove Status Column & Update Rendering**
    - Modify `src/components/TaskTable.tsx`.
    - Action: Remove the `Status` column from the table header and body. Ensure `Key`, `Summary`, and `Type` are the primary metadata columns alongside the Timeline dates.
    - Acceptance: Grep `<th>Status</th>` returns no results in `TaskTable.tsx`.

6.  **Verify CORS Proxy Usage**
    - Audit `src/app/dashboard/page.tsx` again.
    - Action: Ensure NO fetches to `jira.turkcell.com.tr` exist in the client-side code; all must point to `/api/jira/search`.
    - Acceptance: `grep -r "jira.turkcell.com.tr" src/app` returns results ONLY in server-side `route.ts` files or `.env.local`.
